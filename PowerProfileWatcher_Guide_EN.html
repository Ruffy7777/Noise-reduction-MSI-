00 How Does It Work?

How It Works

A watcher runs permanently in the background and reads the NVIDIA dGPU load every 500ms. When stable GPU load is detected (e.g. from a game), it automatically switches to the Gaming profile with full CPU boost. After closing the game, it returns to the Office profile.

🚀 Windows Starts
↓
Task Scheduler launches Watcher (15s delay)
↓
Watcher waits for NVIDIA driver
↓
🔵 Office Profile active  ·  CPU 80%  ·  Boost locked  ·  quiet
↓
Measure GPU load every 500ms
↓
GPU ≥ 5% for ~2 sec
🟢 Gaming Profile
CPU 100% · Boost unlocked · full performance
↓
GPU ≤ 1% for ~6 sec
↑
back to Office
No stable load → no switch
↺ Continuous loop – runs in background

Get-ChildItem "C:\Windows\System32\DriverStore\FileRepository\" -Filter "nvml.dll" -Recurse -ErrorAction SilentlyContinue | Select-Object FullName

Note the GUID of the scheme marked with * (Balanced), e.g. 3f43ac0d-b402-4072-82c5-4e490233737a

03 Create .ps1 Files

A .ps1 file is simply a text file with the .ps1 extension. It can be created with the standard Windows Notepad.

Step 1: Create the C:\Scripts folder

  1. Open Windows Explorer

  2. Click "This PC" on the left → open drive C:

  3. Right-click on empty space → New → Folder

  4. Type: Scripts → Enter

Step 2: Create the script files

Repeat this process three times – for office.ps1, gaming.ps1 and watcher.ps1:

  1. Open the C:\Scripts folder

  2. Right-click on empty space → New → Text Document

  3. Type the filename: office.ps1 (replacing the .txt extension with .ps1)

  4. Windows asks: "Change file extension?" → Click Yes

  5. Double-click the file → it opens in Notepad

  6. Paste the script content (from the next chapters) → Ctrl+S to save

04 office.ps1 – Content

Open C:\Scripts\office.ps1 in Notepad, delete everything and paste the following content.

📁 File Location

File: C:\Scripts\office.ps1

⚠ Customization required

Replace YOUR-GUID-HERE with your GUID from Step 01 (powercfg /list).

office.ps1
# Office Profile: CPU throttled, quiet, energy efficient
$host.UI.RawUI.WindowTitle = "Office Profile"

if (-not ([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)) {
    Write-Host "ERROR: Script must be run as Administrator!" -ForegroundColor Red
    exit 1
}

# Enter your Balanced power scheme GUID here:
$SchemaGUID = "YOUR-GUID-HERE"

powercfg /setactive $SchemaGUID

# Max CPU state: 80% -> Boost locked, max ~2.0 GHz
powercfg -setacvalueindex $SchemaGUID sub_processor PROCTHROTTLEMAX 80
powercfg -setdcvalueindex $SchemaGUID sub_processor PROCTHROTTLEMAX 80

# Min CPU state: 5%
powercfg -setacvalueindex $SchemaGUID sub_processor PROCTHROTTLEMIN 5
powercfg -setdcvalueindex $SchemaGUID sub_processor PROCTHROTTLEMIN 5

# EPP: AC=60 (moderate), DC=80 (battery saving)
powercfg -setacvalueindex $SchemaGUID sub_processor PERFEPP 60
powercfg -setdcvalueindex $SchemaGUID sub_processor PERFEPP 80

powercfg /setactive $SchemaGUID
Write-Host "Office Profile active: 80% Max CPU, Boost locked, EPP 60/80." -ForegroundColor Green

05 gaming.ps1 – Content

This script creates a fresh "Ultimate Performance" scheme on every call. No customization needed – paste as-is.

📁 File Location

File: C:\Scripts\gaming.ps1

gaming.ps1
# Gaming Profile: Full boost, maximum performance
$host.UI.RawUI.WindowTitle = "Gaming Profile"

if (-not ([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)) {
    Write-Host "ERROR: Script must be run as Administrator!" -ForegroundColor Red
    exit 1
}

# Clean up old gaming schemes (prevents accumulation)
$allSchemes = powercfg /list
foreach ($line in $allSchemes) {
    if ($line -match "([a-f0-9\-]{36}).*(Ultimate Performance|Ultimative Leistung)") {
        $oldGUID = $matches[1]
        Write-Host "Deleting old scheme: $oldGUID" -ForegroundColor Yellow
        powercfg -delete $oldGUID
    }
}

# Create new Ultimate Performance scheme
$output = powercfg -duplicatescheme e9a42b02-d5df-448d-aa00-03f14749eb61
$GamingSchemeGUID = ($output | Select-String "GUID.*:\s+(\S+)").Matches.Groups[1].Value
Write-Host "Gaming scheme created: $GamingSchemeGUID" -ForegroundColor Yellow

powercfg -setacvalueindex $GamingSchemeGUID sub_processor PROCTHROTTLEMAX 100
powercfg -setdcvalueindex $GamingSchemeGUID sub_processor PROCTHROTTLEMAX 100
powercfg -setacvalueindex $GamingSchemeGUID sub_processor PROCTHROTTLEMIN 10
powercfg -setdcvalueindex $GamingSchemeGUID sub_processor PROCTHROTTLEMIN 10
powercfg -setacvalueindex $GamingSchemeGUID sub_processor PERFEPP 0
powercfg -setdcvalueindex $GamingSchemeGUID sub_processor PERFEPP 0

powercfg /setactive $GamingSchemeGUID
Write-Host "Gaming Profile active: Boost unlocked, EPP 0." -ForegroundColor Green

06 watcher.ps1 – Content

📁 File Location

File: C:\Scripts\watcher.ps1

⚠ Two places to customize

1. Replace YOUR-NVML-PATH-HERE with the path from Step 01.
2. Replace YOUR-OFFICE-GUID-HERE with your GUID from Step 01.

watcher.ps1
# Wait for NVIDIA driver (nvcontainer OR NVChecker)
while (-not (Get-Process -Name "nvcontainer" -ErrorAction SilentlyContinue) -and `
      -not (Get-Process -Name "NVChecker" -ErrorAction SilentlyContinue)) {
    Start-Sleep -Seconds 1
}
Start-Sleep -Seconds 2

$host.UI.RawUI.WindowTitle = "Power Profile Watcher (NVML dGPU)"

# Ensure Office scheme is active on startup
powercfg /setactive "YOUR-OFFICE-GUID-HERE"

# CUSTOMIZE THESE:
$OfficeScript = "C:\Scripts\office.ps1"
$GamingScript = "C:\Scripts\gaming.ps1"
$NvmlPath     = "YOUR-NVML-PATH-HERE"

# Thresholds (GPU load in %)
$GamingThreshold     = 5   # Above this -> Gaming counter increments
$OfficeThreshold     = 1   # Below this -> Office counter increments
$RequiredGamingReads = 4   # ~2 seconds stable load to switch to Gaming
$RequiredOfficeReads = 12  # ~6 seconds stable idle to switch to Office
$CooldownSeconds     = 10  # Wait time after a switch

$LastSwitch    = (Get-Date).AddSeconds(-$CooldownSeconds)
$GamingCounter = 0
$OfficeCounter  = 0
$CurrentMode   = "Office"

$nvmlCode = @"
using System;
using System.Runtime.InteropServices;
public enum nvmlReturn_t { NVML_SUCCESS = 0 }
[StructLayout(LayoutKind.Sequential)]
public struct nvmlUtilization_t { public uint gpu; public uint memory; }
public static class NVML {
    [DllImport(@"{PATH}", CallingConvention = CallingConvention.Cdecl)]
    public static extern nvmlReturn_t nvmlInit_v2();
    [DllImport(@"{PATH}", CallingConvention = CallingConvention.Cdecl)]
    public static extern nvmlReturn_t nvmlShutdown();
    [DllImport(@"{PATH}", CallingConvention = CallingConvention.Cdecl)]
    public static extern nvmlReturn_t nvmlDeviceGetHandleByIndex_v2(uint index, out IntPtr device);
    [DllImport(@"{PATH}", CallingConvention = CallingConvention.Cdecl)]
    public static extern nvmlReturn_t nvmlDeviceGetUtilizationRates(IntPtr device, out nvmlUtilization_t utilization);
}
"@.Replace("{PATH}", $NvmlPath.Replace("\", "\\"))

Add-Type -TypeDefinition $nvmlCode -ErrorAction Stop

$result = [NVML]::nvmlInit_v2()
if ($result -ne [nvmlReturn_t]::NVML_SUCCESS) { Write-Host "NVML init failed."; exit 1 }

$device = [IntPtr]::Zero
$result  = [NVML]::nvmlDeviceGetHandleByIndex_v2(0, [ref]$device)
if ($result -ne [nvmlReturn_t]::NVML_SUCCESS) { [NVML]::nvmlShutdown() | Out-Null; exit 1 }

Write-Host "Watcher started. Gaming: $RequiredGamingReads reads | Office: $RequiredOfficeReads reads | Cooldown: ${CooldownSeconds}s"

try {
    while ($true) {
        $util   = New-Object nvmlUtilization_t
        $result = [NVML]::nvmlDeviceGetUtilizationRates($device, [ref]$util)
        if ($result -eq [nvmlReturn_t]::NVML_SUCCESS) {
            $gpuLoad            = [int]$util.gpu
            $SecondsSinceSwitch = ((Get-Date) - $LastSwitch).TotalSeconds
            $CooldownActive     = $SecondsSinceSwitch -lt $CooldownSeconds
            if ($gpuLoad -ge $GamingThreshold) { $GamingCounter++; $OfficeCounter = 0 }
            elseif ($gpuLoad -le $OfficeThreshold) { $OfficeCounter++; $GamingCounter = 0 }
            $cd = if ($CooldownActive) { " [Cooldown: $([math]::Round($CooldownSeconds - $SecondsSinceSwitch,1))s]" } else { "" }
            Write-Host "GPU: $gpuLoad% | $CurrentMode | G:$GamingCounter/$RequiredGamingReads O:$OfficeCounter/$RequiredOfficeReads$cd"
            if (-not $CooldownActive) {
                if ($GamingCounter -ge $RequiredGamingReads -and $CurrentMode -ne "Gaming") {
                    Write-Host ">>> Switching to Gaming Profile" -ForegroundColor Green
                    & $GamingScript
                    $CurrentMode = "Gaming"; $LastSwitch = Get-Date
                    $GamingCounter = 0; $OfficeCounter = 0
                }
                if ($OfficeCounter -ge $RequiredOfficeReads -and $CurrentMode -ne "Office") {
                    Write-Host ">>> Switching to Office Profile" -ForegroundColor Cyan
                    & $OfficeScript
                    $CurrentMode = "Office"; $LastSwitch = Get-Date
                    $GamingCounter = 0; $OfficeCounter = 0
                }
            }
        }
        Start-Sleep -Milliseconds 500
    }
} finally { [NVML]::nvmlShutdown() | Out-Null }

07 Task Scheduler Setup

Open Task Scheduler: Win + R → taskschd.msc → Enter.

Task 1: Office on Login (PowerProfileOffice_Start)

  1. Right-click on "Task Scheduler Library" → Create Basic Task

  2. Name: PowerProfileOffice_Start → Next

  3. Trigger: When I log on → Next

  4. Action: Start a program → Next

  5. Program/script: pwsh.exe

  6. Arguments: -ExecutionPolicy Bypass -WindowStyle Hidden -File "C:\Scripts\office.ps1"

  7. Start in: C:\Program Files\PowerShell\7

  8. Finish → check "Open the Properties dialog" → Finish

  9. Tab "General" → check "Run with highest privileges"

  10. Tab "Triggers" → double-click trigger → enable Delay → type 15 seconds → OK → OK

Task 2: Watcher at Startup (PowerProfileWatcher_PS7)

  1. Right-click on "Task Scheduler Library" → Create Basic Task

  2. Name: PowerProfileWatcher_PS7 → Next

  3. Trigger: When the computer starts → Next

  4. Action: Start a program → Next

  5. Program/script: C:\PROGRA~1\PowerShell\7\pwsh.exe (short path – no space issues)

  6. Arguments: -ExecutionPolicy Bypass -WindowStyle Hidden -File "C:\Scripts\watcher.ps1"

  7. Finish → check "Open the Properties dialog" → Finish

  8. Tab "General" → change user to your own account (not SYSTEM) → check "Run with highest privileges"

  9. Tab "Triggers" → double-click → enable Delay → type 15 seconds → OK → OK

08 Set Execution Policy

Run this once in an Admin PowerShell 7. Without this setting, Windows will refuse to run the scripts.

PowerShell 7 (once – Admin)
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope LocalMachine

Confirm with Y. This setting is saved permanently.

✓ After this, all scripts can be executed without any restrictions.

10 Troubleshooting

Watcher doesn't start / black window

NVIDIA driver process is not running yet. Check which processes are active:

PowerShell
Get-Process | Where-Object { $_.Name -like "*nv*" } | Format-Table Name, Id

Script won't run (security error)

Execution Policy is missing → run Step 08.

CPU won't boost above base clock

PowerShell
powercfg /list
# In Gaming mode, "Ultimate Performance" must be marked with *

CPU stuck at very low MHz (e.g. 1.4 GHz)

PowerShell
powercfg -query scheme_current sub_processor PROCTHROTTLEMAX
# Should be 0x50 (80%) in Office or 0x64 (100%) in Gaming

NVIDIA services not running after reboot

PowerShell (Admin)
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\NvContainerLocalSystem" -Name "Start" -Value 2
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\NVDisplay.ContainerLocalSystem" -Name "Start" -Value 2
# Then restart

After a driver update: NVML path has changed

PowerShell
Get-ChildItem "C:\Windows\System32\DriverStore\FileRepository\" -Filter "nvml.dll" -Recurse | Select-Object FullName

Update the new path in C:\Scripts\watcher.ps1 at the $NvmlPath variable.
