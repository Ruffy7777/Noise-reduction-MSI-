Power Profile Watcher

Automatische CPU-Leistungssteuerung für Windows 11. Im Office-Modus leise und sparsam, beim Gaming volle Boost-Performance – vollständig automatisch.

00 Wie funktioniert das System?

Funktionsweise

Ein dauerhaft im Hintergrund laufender Watcher liest alle 500ms die GPU-Auslastung der NVIDIA dGPU. Sobald stabile Last erkannt wird (z.B. durch ein Spiel), wechselt er automatisch ins Gaming-Profil mit vollem CPU-Boost. Nach dem Schließen des Spiels kehrt er ins Office-Profil zurück.

NVML-Pfad herausfinden

Öffne PowerShell 7 als Administrator und führe aus:

PowerShell 7 (Admin)
Get-ChildItem "C:\Windows\System32\DriverStore\FileRepository\" -Filter "nvml.dll" -Recurse -ErrorAction SilentlyContinue | Select-Object FullName

Beispielausgabe: C:\Windows\System32\DriverStore\FileRepository\nvmii.inf_amd64_XXXXXXXX\nvml.dll – diesen Pfad notieren.

PowerShell
powercfg /list

Die GUID des mit * markierten Schemas notieren, z.B. 3f43ac0d-b402-4072-82c5-4e490233737a

03 .ps1 Dateien erstellen

Eine .ps1-Datei ist eine einfache Textdatei mit der Endung .ps1. Sie kann mit dem normalen Windows-Editor (Notepad) erstellt werden.

Schritt 1: Ordner C:\Scripts erstellen

  1. Öffne den Windows Explorer

  2. Klicke links auf „Dieser PC" → öffne Laufwerk C:

  3. Rechtsklick auf eine leere Stelle → Neu → Ordner

  4. Name eingeben: Scripts → Enter

Schritt 2: Script-Dateien anlegen

Diesen Vorgang dreimal wiederholen – für office.ps1, gaming.ps1 und watcher.ps1:

  1. Öffne den Ordner C:\Scripts

  2. Rechtsklick auf eine leere Stelle → Neu → Textdokument

  3. Dateiname direkt eingeben: office.ps1 (die .txt-Endung wird durch .ps1 ersetzt)

  4. Windows fragt: „Dateinamenerweiterung ändern?" → Ja klicken

  5. Doppelklick auf die Datei → öffnet sich im Editor

  6. Script-Inhalt einfügen (aus den nächsten Kapiteln kopieren) → Strg+S zum Speichern

04 office.ps1 – Inhalt einfügen

Öffne C:\Scripts\office.ps1 mit dem Editor, lösche alles was drin ist und füge folgenden Inhalt ein.

📁 Speicherort

Datei: C:\Scripts\office.ps1

⚠ Anpassen erforderlich

Ersetze DEINE-GUID-HIER-EINTRAGEN mit deiner GUID aus Schritt 01 (powercfg /list).

office.ps1
# Office-Profil: CPU gedrosselt, leise, energiesparend
$host.UI.RawUI.WindowTitle = "Office Profil"

if (-not ([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)) {
    Write-Host "FEHLER: Script muss als Administrator ausgefuehrt werden!" -ForegroundColor Red
    exit 1
}

# GUID des Ausbalanciert-Schemas hier eintragen:
$SchemaGUID = "DEINE-GUID-HIER-EINTRAGEN"

powercfg /setactive $SchemaGUID

# Maximaler CPU-Zustand: 80% -> Boost gesperrt, max ~2,0 GHz
powercfg -setacvalueindex $SchemaGUID sub_processor PROCTHROTTLEMAX 80
powercfg -setdcvalueindex $SchemaGUID sub_processor PROCTHROTTLEMAX 80

# Minimaler CPU-Zustand: 5%
powercfg -setacvalueindex $SchemaGUID sub_processor PROCTHROTTLEMIN 5
powercfg -setdcvalueindex $SchemaGUID sub_processor PROCTHROTTLEMIN 5

# EPP: AC=60 (moderat), DC=80 (Akku schonen)
powercfg -setacvalueindex $SchemaGUID sub_processor PERFEPP 60
powercfg -setdcvalueindex $SchemaGUID sub_processor PERFEPP 80

powercfg /setactive $SchemaGUID
Write-Host "Office Profil aktiv: 80% Max CPU, Boost gesperrt, EPP 60/80." -ForegroundColor Green

05 gaming.ps1 – Inhalt einfügen

Dieses Script erstellt bei jedem Aufruf ein frisches „Ultimative Leistung"-Schema. Kein Anpassen nötig – einfach so einfügen.

📁 Speicherort

Datei: C:\Scripts\gaming.ps1

gaming.ps1
# Gaming-Profil: Voller Boost, maximale Performance
$host.UI.RawUI.WindowTitle = "Gaming Profil"

if (-not ([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)) {
    Write-Host "FEHLER: Script muss als Administrator ausgefuehrt werden!" -ForegroundColor Red
    exit 1
}

# Alte Gaming-Schemas aufräumen (verhindert Anhäufung)
$alleSchemas = powercfg /list
foreach ($zeile in $alleSchemas) {
    if ($zeile -match "([a-f0-9\-]{36}).*Ultimative Leistung") {
        $alteGUID = $matches[1]
        Write-Host "Altes Schema wird geloescht: $alteGUID" -ForegroundColor Yellow
        powercfg -delete $alteGUID
    }
}

# Neues "Ultimative Leistung" Schema erstellen
$output = powercfg -duplicatescheme e9a42b02-d5df-448d-aa00-03f14749eb61
$GamingSchemaGUID = ($output | Select-String "GUID.*:\s+(\S+)").Matches.Groups[1].Value
Write-Host "Gaming-Schema erstellt: $GamingSchemaGUID" -ForegroundColor Yellow

powercfg -setacvalueindex $GamingSchemaGUID sub_processor PROCTHROTTLEMAX 100
powercfg -setdcvalueindex $GamingSchemaGUID sub_processor PROCTHROTTLEMAX 100
powercfg -setacvalueindex $GamingSchemaGUID sub_processor PROCTHROTTLEMIN 10
powercfg -setdcvalueindex $GamingSchemaGUID sub_processor PROCTHROTTLEMIN 10
powercfg -setacvalueindex $GamingSchemaGUID sub_processor PERFEPP 0
powercfg -setdcvalueindex $GamingSchemaGUID sub_processor PERFEPP 0

powercfg /setactive $GamingSchemaGUID
Write-Host "Gaming Profil aktiv: Boost freigegeben, EPP 0." -ForegroundColor Green

06 watcher.ps1 – Inhalt einfügen

📁 Speicherort

Datei: C:\Scripts\watcher.ps1

⚠ Zwei Stellen anpassen

1. Ersetze DEIN-NVML-PFAD-HIER mit dem Pfad aus Schritt 01.
2. Ersetze DEINE-OFFICE-GUID-HIER mit der GUID aus Schritt 01.

watcher.ps1
# Warten bis NVIDIA Treiber bereit ist (nvcontainer ODER NVChecker)
while (-not (Get-Process -Name "nvcontainer" -ErrorAction SilentlyContinue) -and `
      -not (Get-Process -Name "NVChecker" -ErrorAction SilentlyContinue)) {
    Start-Sleep -Seconds 1
}
Start-Sleep -Seconds 2

$host.UI.RawUI.WindowTitle = "Power Profile Watcher (NVML dGPU)"

# Office-Schema beim Start sicherstellen
powercfg /setactive "DEINE-OFFICE-GUID-HIER"

# HIER ANPASSEN:
$OfficeScript = "C:\Scripts\office.ps1"
$GamingScript = "C:\Scripts\gaming.ps1"
$NvmlPath     = "DEIN-NVML-PFAD-HIER"

# Schwellenwerte (GPU-Last in %)
$GamingThreshold     = 5   # Ab diesem Wert -> Gaming wird gezählt
$OfficeThreshold     = 1   # Unter diesem Wert -> Office wird gezählt
$RequiredGamingReads = 4   # ~2 Sekunden stabile Last für Gaming-Wechsel
$RequiredOfficeReads = 12  # ~6 Sekunden stabiles Idle für Office-Wechsel
$CooldownSeconds     = 10  # Wartezeit nach einem Wechsel

$LastSwitch    = (Get-Date).AddSeconds(-$CooldownSeconds)
$GamingCounter = 0
$OfficeCounter  = 0
$CurrentMode   = "Office"

$nvmlCode = @"
using System;
using System.Runtime.InteropServices;
public enum nvmlReturn_t { NVML_SUCCESS = 0 }
[StructLayout(LayoutKind.Sequential)]
public struct nvmlUtilization_t { public uint gpu; public uint memory; }
public static class NVML {
    [DllImport(@"{PATH}", CallingConvention = CallingConvention.Cdecl)]
    public static extern nvmlReturn_t nvmlInit_v2();
    [DllImport(@"{PATH}", CallingConvention = CallingConvention.Cdecl)]
    public static extern nvmlReturn_t nvmlShutdown();
    [DllImport(@"{PATH}", CallingConvention = CallingConvention.Cdecl)]
    public static extern nvmlReturn_t nvmlDeviceGetHandleByIndex_v2(uint index, out IntPtr device);
    [DllImport(@"{PATH}", CallingConvention = CallingConvention.Cdecl)]
    public static extern nvmlReturn_t nvmlDeviceGetUtilizationRates(IntPtr device, out nvmlUtilization_t utilization);
}
"@.Replace("{PATH}", $NvmlPath.Replace("\", "\\"))

Add-Type -TypeDefinition $nvmlCode -ErrorAction Stop

$result = [NVML]::nvmlInit_v2()
if ($result -ne [nvmlReturn_t]::NVML_SUCCESS) { Write-Host "NVML Init fehlgeschlagen."; exit 1 }

$device = [IntPtr]::Zero
$result  = [NVML]::nvmlDeviceGetHandleByIndex_v2(0, [ref]$device)
if ($result -ne [nvmlReturn_t]::NVML_SUCCESS) { [NVML]::nvmlShutdown() | Out-Null; exit 1 }

Write-Host "Watcher gestartet. Gaming: $RequiredGamingReads Reads | Office: $RequiredOfficeReads Reads | Cooldown: ${CooldownSeconds}s"

try {
    while ($true) {
        $util   = New-Object nvmlUtilization_t
        $result = [NVML]::nvmlDeviceGetUtilizationRates($device, [ref]$util)
        if ($result -eq [nvmlReturn_t]::NVML_SUCCESS) {
            $gpuLoad            = [int]$util.gpu
            $SecondsSinceSwitch = ((Get-Date) - $LastSwitch).TotalSeconds
            $CooldownActive     = $SecondsSinceSwitch -lt $CooldownSeconds
            if ($gpuLoad -ge $GamingThreshold) { $GamingCounter++; $OfficeCounter = 0 }
            elseif ($gpuLoad -le $OfficeThreshold) { $OfficeCounter++; $GamingCounter = 0 }
            $cd = if ($CooldownActive) { " [Cooldown: $([math]::Round($CooldownSeconds - $SecondsSinceSwitch,1))s]" } else { "" }
            Write-Host "GPU: $gpuLoad% | $CurrentMode | G:$GamingCounter/$RequiredGamingReads O:$OfficeCounter/$RequiredOfficeReads$cd"
            if (-not $CooldownActive) {
                if ($GamingCounter -ge $RequiredGamingReads -and $CurrentMode -ne "Gaming") {
                    Write-Host ">>> Gaming Profil" -ForegroundColor Green
                    & $GamingScript
                    $CurrentMode = "Gaming"; $LastSwitch = Get-Date
                    $GamingCounter = 0; $OfficeCounter = 0
                }
                if ($OfficeCounter -ge $RequiredOfficeReads -and $CurrentMode -ne "Office") {
                    Write-Host ">>> Office Profil" -ForegroundColor Cyan
                    & $OfficeScript
                    $CurrentMode = "Office"; $LastSwitch = Get-Date
                    $GamingCounter = 0; $OfficeCounter = 0
                }
            }
        }
        Start-Sleep -Milliseconds 500
    }
} finally { [NVML]::nvmlShutdown() | Out-Null }

07 Aufgabenplanung einrichten

Öffne die Aufgabenplanung: Win + R → taskschd.msc → Enter.

Aufgabe 1: Office beim Start (PowerProfileOffice_Start)

  1. Rechtsklick auf „Aufgabenplanungsbibliothek" → Einfache Aufgabe erstellen

  2. Name: PowerProfileOffice_Start → Weiter

  3. Trigger: Bei Anmeldung → Weiter

  4. Aktion: Programm starten → Weiter

  5. Programm/Skript: pwsh.exe

  6. Argumente: -ExecutionPolicy Bypass -WindowStyle Hidden -File "C:\Scripts\office.ps1"

  7. Starten in: C:\Program Files\PowerShell\7

  8. Fertig stellen → Häkchen bei „Eigenschaften öffnen" → Fertig stellen

  9. Tab „Allgemein" → „Mit höchsten Berechtigungen ausführen" anhaken

  10. Tab „Trigger" → Doppelklick → Verzögerung aktivieren → 15 Sekunden → OK → OK

Aufgabe 2: Watcher beim Systemstart (PowerProfileWatcher_PS7)

  1. Rechtsklick auf „Aufgabenplanungsbibliothek" → Einfache Aufgabe erstellen

  2. Name: PowerProfileWatcher_PS7 → Weiter

  3. Trigger: Beim Start des Computers → Weiter

  4. Aktion: Programm starten → Weiter

  5. Programm/Skript: C:\PROGRA~1\PowerShell\7\pwsh.exe (Kurzpfad – kein Leerzeichen-Problem)

  6. Argumente: -ExecutionPolicy Bypass -WindowStyle Hidden -File "C:\Scripts\watcher.ps1"

  7. Fertig stellen → Häkchen bei „Eigenschaften öffnen" → Fertig stellen

  8. Tab „Allgemein" → Benutzer auf deinen Account ändern (nicht SYSTEM) → „Mit höchsten Berechtigungen ausführen" anhaken

  9. Tab „Trigger" → Doppelklick → Verzögerung: 15 Sekunden → OK → OK

08 Execution Policy setzen

Einmalig in einer Admin-PowerShell 7 ausführen. Ohne diese Einstellung verweigert Windows die Ausführung der Scripts.

PowerShell 7 (einmalig – Admin)
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope LocalMachine

Mit J bestätigen. Diese Einstellung bleibt dauerhaft gespeichert.

✓ Nach dieser Einstellung können alle Scripts ohne Einschränkungen ausgeführt werden.

10 Troubleshooting

Watcher startet nicht / schwarzes Fenster

NVIDIA-Treiberprozess läuft noch nicht. Prüfen welcher Prozess aktiv ist:

PowerShell
Get-Process | Where-Object { $_.Name -like "*nv*" } | Format-Table Name, Id

Script wird nicht ausgeführt (Sicherheitsfehler)

Execution Policy fehlt → Schritt 08 ausführen.

CPU geht nicht über Basistakt (kein Boost)

PowerShell
powercfg /list
# Im Gaming-Modus muss "Ultimative Leistung" mit * markiert sein

CPU bleibt bei sehr niedrigen MHz (z.B. 1,4 GHz)

PowerShell
powercfg -query scheme_current sub_processor PROCTHROTTLEMAX
# Sollte 0x50 (80%) im Office oder 0x64 (100%) im Gaming sein

NVIDIA-Dienste laufen nicht nach Neustart

PowerShell (Admin)
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\NvContainerLocalSystem" -Name "Start" -Value 2
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\NVDisplay.ContainerLocalSystem" -Name "Start" -Value 2
# Danach neu starten

Nach Treiberupdate: NVML-Pfad hat sich geändert

PowerShell
Get-ChildItem "C:\Windows\System32\DriverStore\FileRepository\" -Filter "nvml.dll" -Recurse | Select-Object FullName

Den neuen Pfad in C:\Scripts\watcher.ps1 bei $NvmlPath aktualisieren.
